# Security Policy

Thanks for helping keep Scuba Steve AI safe.

## Supported Versions

We support security fixes for the latest production deployment and the default branch.

| Version | Supported |
|--------|-----------|
| Production (Vercel) | ✅ |
| `main` branch | ✅ |
| Older commits / forks | ❌ |

## Reporting a Vulnerability

Please report security issues privately.

**Email:** steve@scubasteve.rocks  
**Subject:** `Security report: <short summary>`  

Include:
- What you found + steps to reproduce
- Affected URL(s) / endpoint(s)
- Screenshots/logs if available
- Your suggested fix (optional)

### Response timeline
- **Acknowledgement:** within 72 hours
- **Status update:** within 7 days
- **Fix + release:** as fast as possible depending on severity

## Safe Harbor

If you:
- Make a good-faith effort to avoid privacy violations and data destruction
- Do not use social engineering, DDoS, or physical attacks
- Do not access or modify other users’ data

…we will not pursue legal action for your research.

## Scope

### In scope
- https://scubasteve.rocks
- https://www.scubasteve.rocks
- https://scuba-steve-rocks.vercel.app
- Firebase-backed APIs used by this app (Firestore/Storage rules)

### Out of scope
- Third-party services (Stripe, Google, Vercel platform issues)
- Rate limiting / spam unless it causes direct security impact
